Usage

1. Add dependency on your pom.xml file {% highlight java %} io.github.simple4tests interactions-webdriver ... {% endhighlight %} 2. Instanciate a WebDriverInteractions object {% highlight java %} WebDriver driver; ... driver = new ChromeDriver(); ... WebDriverInteractions wdi = new WebDriverInteractions(driver); {% endhighlight %} 3a. Use WebDriver Interactions functions {% highlight java %} wdi.browser.navigateTo(myUrl); wdi.click(buttonLocator); wdi.set(inputLocator, value); ... {% endhighlight %} 3b. Use native WebDriver functions {% highlight java %} wdi.driver.getCurrentUrl(); wdi.driver. ... {% endhighlight %}

Recommended implementation

Even WebDriverInteractions object can be used straight forward as shown previuosly, the recommendation is to create in your project a class Wdi that extends WebdriverInteractions. This way, you will be able to easely customize WebDriverInteractions creating your custom technical actions based on WebDriverInteractions.

{% highlight java %} public class Wdi extends WebDriverInteractions { public Wdi(WebDriver driver) { super(driver); } // custom technical action to select a value within a custom dropdown public void selectInList(By by, String value) {... click(...); ...; click(...); ...} } {% endhighlight %}

Then instanciate your newly class Wdi and you are ready to play with WebDriverInteractions functions as well as your custom functions. All ot them from a single object.

{% highlight java %} WebDriver driver; ... driver = new ChromeDriver(); ... Wdi wdi = new Wdi(driver); wdi.browser.navigateTo(...); wdi.click(...); wdi.set(...); wdi.selectInList(by, value); ... {% endhighlight %}
